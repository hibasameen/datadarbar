# Pakistan Census Explorer (District-level Education & Health)

An interactive, district-level map for exploring key indicators from the Pakistan Bureau of Statistics (PBS) 2023 Census (district tables 12 & 13).

## Features
- Choropleth by district with hover, click, and tooltips
- Switchable indicators (literacy, attendance, disability, health access)
- Province filter and district search
- CSV download of the current view
- Pure static frontend (Leaflet, no build step) — deployable to GitHub Pages
- Lightweight ETL scripts to scrape and parse PBS PDFs into a single JSON for the app

## Repo structure
```
pbs-census-map/
  app/
    index.html
    assets/css/styles.css
    assets/js/app.js
    data/
      pakistan_districts_province_boundries.geojson   # put your GeoJSON here
      pbs_district_indicators.json                     # generated by ETL
  etl/
    scrape_pbs.py
    parse_pdfs.py
    build_dataset.py
```

## How to use

### 1) Prepare GeoJSON
Copy your district GeoJSON into `app/data/` and name it `pakistan_districts_province_boundries.geojson`. If you prefer a different name, update `GEOJSON_PATH` at the bottom of `index.html`.

### 2) Get data from PBS (once, or when updated)
```
cd etl
python scrape_pbs.py --out ./downloads
python parse_pdfs.py --indir ./downloads --out ./district_indicators.csv
python build_dataset.py --csv ./district_indicators.csv --out ../app/data/pbs_district_indicators.json
```
> Note: the PDF parser uses heuristics and may need tweaks depending on the exact layout of PBS tables. Open `parse_pdfs.py` and adjust the regex in `parse_table_text()` if some districts are missed.

### 3) Run locally
Just open `app/index.html` in your browser, or serve the `app/` folder via a simple server:
```
# from the app directory
python -m http.server 8000
```

### 4) Deploy to GitHub Pages
- Create a new GitHub repo, commit the entire `pbs-census-map` folder.
- In Settings → Pages, set the branch to `main` (or `master`) and folder `/app`.
- Wait ~1–2 minutes, your site will be live at `https://<username>.github.io/<repo>/`.

### 5) Deploy to Railway (optional, if you want server-side scraping)
This is a static site; Railway isn’t required. If you want scheduled scraping on Railway:
- Add a `railway.toml` and a simple Python service that runs `etl/` on a cron, pushes JSON to object storage, and serves it via a tiny FastAPI.
- Point the frontend `DATA_PATH` to that endpoint. (Ask me and I’ll wire this up.)

## Data notes
- This project expects the PBS district tables to include: literacy (total/male/female), school attendance, disability rate, and “no health facility within 5km”. If your chosen indicators differ, update:
  - `etl/parse_pdfs.py` (column mapping)
  - `app/assets/js/app.js` (`niceLabel`, indicator list)
- District name matching is by normalized name (lowercase, alphanumeric). If your GeoJSON uses different naming, add a crosswalk or aliases.
